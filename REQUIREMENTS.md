# 가계부 프로그램 요구사항 정의서

## 1. 개요

### 1.1 목적
개인 및 가정의 수입과 지출을 체계적으로 기록·분류·분석하여, 사용자가 자신의 재무 상태를 한눈에 파악하고 합리적인 소비 습관을 형성하도록 돕는 가계부 프로그램을 개발한다.

### 1.2 대상 사용자
- 개인 재무 관리를 시작하려는 일반 사용자
- 가정의 예산을 관리하는 가구 구성원
- 소비 패턴 분석과 절약 목표 설정이 필요한 사용자

### 1.3 핵심 가치
- **간편함**: 빠르고 부담 없는 입력 경험
- **정확함**: 신뢰할 수 있는 집계와 잔액 관리
- **통찰**: 데이터 기반의 소비 분석과 시각화

## 4. 데이터 모델 (개념 설계)

### 4.1 주요 엔티티

**거래 (Transaction)**
- `id`: 고유 식별자
- `type`: 수입 / 지출
- `amount`: 금액
- `date`: 거래 일시
- `category_id`: 카테고리 참조
- `payment_method`: 결제수단
- `memo`: 메모
- `created_at` / `updated_at`: 생성·수정 시각

**카테고리 (Category)**
- `id`: 고유 식별자
- `name`: 이름
- `type`: 수입 / 지출
- `parent_id`: 상위 카테고리 (계층 구조 시)
- `icon` / `color`: 표시용 아이콘·색상

**예산 (Budget)**
- `id`: 고유 식별자
- `category_id`: 대상 카테고리 (전체 예산은 null)
- `period`: 적용 기간 (예: 2026-06)
- `amount`: 예산 금액

**계좌/자산 (Account)** *(선택)*
- `id`, `name`, `type`(현금/카드/계좌), `balance`

### 4.2 관계
- 하나의 카테고리는 다수의 거래를 가진다 (1:N)
- 하나의 카테고리는 다수의 예산을 가진다 (1:N)
- 하나의 계좌는 다수의 거래를 가진다 (1:N)

## 5. 화면 구성 (UI 흐름)

1. **대시보드(홈)**: 이번 달 요약, 예산 소진율, 최근 거래
2. **거래 입력**: 빠른 입력 폼 (금액 → 분류 → 메모 → 저장)
3. **거래 내역**: 목록/달력 뷰, 검색·필터
4. **통계**: 차트 기반 분석 화면
5. **예산**: 예산 설정 및 진행 현황
6. **설정**: 카테고리, 백업, 보안, 테마

## 7. 개발 단계 (로드맵)

### 1단계 — MVP (최소 기능 제품)
- 거래 CRUD, 기본 카테고리, 월별 요약
- 로컬 데이터 저장

### 2단계 — 분석 강화
- 카테고리별 차트, 예산 관리, 검색·필터

### 3단계 — 편의·확장
- 반복 거래, 데이터 백업/내보내기, 알림

### 4단계 — 온라인/협업
- 계정·동기화, 가계부 공유, 클라우드 백업

## 8. 향후 고려 사항
- 영수증 사진 첨부 및 OCR 자동 입력
- 은행/카드 내역 자동 연동 (오픈뱅킹 등)
- 다국어 및 다중 통화 지원
