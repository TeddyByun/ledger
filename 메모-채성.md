# 채성 메모

## 외부 연결
https://ledger.so4.kr/  
서버 실행하면 여기서 연결되게 해 놨어요(외부 3001포트로 연결)

## 도커 볼륨 주소
`/var/lib/docker/volumes/coder-9273aa84-f606-4ca6-9913-33836d844b98-home/_data/ledger`

## 도커 익스포트 (클로드 응답 복사해옴)
```
teddy 계정에서 아래 명령들을 sudo 없이 쓸 수 있습니다.

1. 익스포트 (백업)

먼저 정합성을 위해 워크스페이스를 멈추는 걸 권합니다 (Coder UI에서 Stop, 또는 coder stop householdledger).

VOL=coder-9273aa84-f606-4ca6-9913-33836d844b98-home

docker run --rm \
  -v $VOL:/data:ro \
  -v $PWD:/backup \
  alpine tar czf /backup/ledger-home.tar.gz --numeric-owner -C /data .

- --numeric-owner → 컨테이너 내 coder(uid 1000) 소유권 보존. 빠뜨리면 복원 시 파일 주인이 깨집니다.
- 3.7G라 몇 분 걸리고, 압축 후 대략 1.5~2G 예상

2. 임포트 (복원)

⚠️ 볼륨 이름에 워크스페이스 UUID가 박혀 있습니다 (coder-<uuid>-home). 그래서 다른 서버나 새 워크스페이스로 복원할 땐 Coder에서 워크스페이스를 먼저 만든 뒤, 새로 생긴 볼륨 이름을 찾아서 거기에 풀어야 합니다.

# 1) Coder UI에서 워크스페이스 생성 (docker 템플릿) → 그 다음 새 볼륨 이름 확인
docker inspect coder-<유저>-<워크스페이스> --format '{{range .Mounts}}{{.Name}}{{end}}'

# 2) 워크스페이스 Stop 후 복원
NEWVOL=<위에서 나온 이름>
docker run --rm \
  -v $NEWVOL:/data \
  -v $PWD:/backup \
  alpine tar xzf /backup/ledger-home.tar.gz --numeric-owner -C /data

기존 볼륨에 덮어쓸 땐 컨테이너가 반드시 멈춰 있어야 합니다.

3. 파일 업로드

docker cp ./내파일 coder-teddy-householdledger:/home/coder/
docker exec coder-teddy-householdledger chown coder:coder /home/coder/내파일

docker cp는 root 소유로 복사되므로 chown이 거의 항상 필요합니다. 디렉토리면 chown -R.

반대로 꺼내기:
docker cp coder-teddy-householdledger:/home/coder/ledger/파일 ./

---
기억해두실 것

이미지 레이어는 백업에 안 들어갑니다. 위 방법은 /home/coder 볼륨만 담습니다. 컨테이너 안에서 sudo apt install로 깔았던 것들은 복원해도 안 돌아옵니다.

그런 건 템플릿(~/projects/coder/templates/docker/main.tf)에 적어두고 rebuild하는 게 Coder의 설계 의도예요. 지금은 기본 이미지 그대로라 문제없지만, 워크스페이스 안에서 패키지를 설치해 쓰고 계시다면 그 목록을 템플릿에 반영해두는 걸 권합니다.

용량 참고: 3.7G 중 .local 2.3G + .cache 629M은 대부분 재생성 가능한 캐시입니다. 소스만 필요하면 -C /data ledger로 848M만 뽑는 게 훨씬 빠릅니다.
```
