## 1. 시각화 상세 (Recharts)

### 1.1 화면별 차트 인벤토리 (폼은 "데이터의 일" 기준으로 선택)

| 위치 | 차트 | 데이터 소스 | 폼 선택 근거 |
|------|------|-------------|-------------|
| 대시보드 | **수입/지출/순액 스탯 타일 3종** | `monthly_summary(ym)` | 단일 헤드라인 숫자 → 차트 아님, 스탯 타일 |
| 대시보드 | **분류 도넛**(이번 달 지출 구성) | `monthly_category_stat(ym)` | 부분-전체 구성 → 도넛(상위 N + 기타) |
| 대시보드 | **예산 소진율 미터**(카테고리별 막대) | `/budgets/status?ym` | 진행률(0~100%+) → 가로 막대 미터 |
| 통계 | **월별 수입 vs 지출 추세** | `monthly_summary` 최근 12개월 | 시간 변화 → 그룹 세로막대 또는 라인 2계열 |
| 통계 | **분류별 지출 추세** | `monthly_category_stat` 다월 | 시간×구성 → 누적 영역(상위 N) |
| 통계 | **결제수단별 비중** | `monthly_payment_stat(ym)` | 구성 비교 → 가로 막대(정렬) |
| 통계 | **수입처별 비중** | `monthly_source_stat(ym)` | 구성 비교 → 가로 막대(정렬) |

> **스탯 타일 우선**: 순액·이번달 합계처럼 값 하나가 핵심이면 차트로 만들지 않고 큰 숫자 + 전월 대비 델타로 표기.

### 1.2 색 규약 (일 기준 배정 — 색이 마지막)

- **극성(수입/지출)**: 수입=파랑 계열, 지출=빨강 계열(FRONTEND §10 확정). 이는 **상태/극성 색**이므로 카테고리 색으로 재사용 금지.
- **분류(카테고리, identity)**: **고정 순서 팔레트**를 대분류 코드(01~12)에 **1:1 고정 배정**(정렬·필터로 재도색 금지 — 색은 엔티티를 따른다). 12개 대분류가 팔레트(≤8색) 초과 → **상위 7 + "기타" 집계**로 접어 표기(도넛·추세 모두).
- **순차(단일 지표 강도, 예: 히트맵형 요일별 지출)**: 단일 hue 명→암. 무지개 금지.
- **다크모드**: 라이트에서 자동 반전 금지 — 동일 램프에서 **다크 표면 기준으로 별도 스텝 산출**.
- **구현 시 필수**: 카테고리 팔레트는 배포 전 **색맹 안전 검증기 실행**(인접쌍 CVD 분리·대비). 검증 통과 전 색 확정 금지. 색만으로 식별 금지 → 범례 + (≤4계열) 직접 라벨 병행.

### 1.3 차트 공통 규칙 (Recharts 매핑)

- **단일 축 원칙**: 이중 y축(수입 금액 vs 건수 등 스케일 다른 2지표) **금지**. 필요 시 **차트 2개** 또는 공통 기준 지수화. (가장 흔한 실수)
- **마크**: 얇은 마크, 막대 데이터끝 4px 라운드+баseline 고정, 라인 2px, 마커 ≥8px, 인접/누적 채움 사이 2px 표면 갭.
- **범례/라벨**: 2계열↑ 범례 상시, 단일 계열은 제목이 명명(범례 박스 없음). 모든 점에 숫자 라벨 금지 — 선택적 직접 라벨.
- **그리드/축**: 후퇴색(muted), 텍스트는 텍스트 토큰 색(계열색 사용 금지).
- **인터랙션**(기본 탑재): 라인/영역=십자선+툴팁, 막대/도넛=마크별 hover 툴팁. 필터(기간·차원)는 차트 위 한 줄. → §1.4.
- **포맷**: 축·툴팁 금액 `Intl.NumberFormat('ko-KR', {notation:'compact'})`(예: ₩1.3M), 툴팁 상세는 원화 전체 표기(FRONTEND §11).

### 1.4 인터랙션 상세

- **기간 선택**: 통계 화면 상단 기간 컨트롤(최근 3/6/12개월·YYYY-MM 범위). 변경 시 관련 쿼리(`['stats','monthly',range]`) 무효화.
- **드릴다운**: 분류 도넛 세그먼트 클릭 → 해당 분류의 거래 목록(`/transactions?categoryCode=`)으로 이동(대분류→소분류→거래).
- **툴팁**: 값 + 전체 대비 %(구성 차트) 또는 전월 대비 Δ(추세 차트).
- **크로스 필터**: 결제수단 막대 hover 시 대응 요약 강조(후순위).

### 1.5 접근성·대체표현

- 모든 차트에 **표(table) 뷰 토글** 제공(스크린리더·인쇄·색 불가 상황). Recharts 데이터와 동일 소스.
- CVD/포싱컬러 대비 텍스처(45°/135°) 옵션. 상태 색(예산 초과=위험)은 **아이콘+라벨 동반**(색 단독 금지).

## 2. 상태 UX 상세 (로딩 · 에러 · 빈 · 알림)

FRONTEND §12의 방향을 **컴포넌트 규격**으로 확정. 4상태를 모든 데이터 화면이 일관되게 처리.

### 2.1 로딩 (Skeleton)

| 영역 | 규격 |
|------|------|
| 목록/표 | 행 스켈레톤 N개(pageSize 근사), 실제 열 폭 유지 |
| 카드/스탯 타일 | 제목·숫자 자리 블록, 높이 고정(레이아웃 시프트 방지) |
| 차트 | 축 프레임 + 회색 플롯 영역 shimmer |
| 규칙 | TanStack Query `isPending` → 스켈레톤. `isFetching`(백그라운드 재검증)은 상단 얇은 프로그레스바만, 콘텐츠 유지 |

### 2.2 빈 상태 (Empty) — 행동 유도

| 화면 | 메시지 | 1차 CTA |
|------|--------|---------|
| 거래 목록(첫 사용) | "아직 거래가 없어요" | **빠른 입력** / **명세서 업로드** |
| 검토 화면 | "검토할 항목이 없습니다" | 대시보드로 |
| 통계(데이터 부족) | "이번 달 데이터가 아직 없어요" | 거래 추가 |
| 예산(미설정) | "예산을 설정하면 소진율을 볼 수 있어요" | 예산 설정 |

- 빈 상태 ≠ 에러. 일러스트(경량)+한 줄 안내+CTA 1개(과하지 않게).

### 2.3 에러 상태

- **인라인 재시도**: 쿼리 실패 시 카드/섹션 단위 "다시 시도" 버튼(`refetch`). 전체 화면 교체 지양.
- **에러 봉투 연동**([API_CONVENTIONS §2](API_CONVENTIONS_DESIGN.md)): `error.code` 기준 분기 —
  | code | UI 처리 |
  |------|---------|
  | `UNAUTHENTICATED` | refresh 자동 시도 → 실패 시 `/login` |
  | `FORBIDDEN` | "권한이 없습니다"(viewer 쓰기 등), 액션 숨김/비활성 |
  | `VALIDATION_FAILED` | 폼 필드별 인라인 에러(`details[].field`) |
  | `RATE_LIMITED` | "잠시 후 다시 시도" + 재시도 지연 |
  | `INTERNAL` | 일반 오류 + traceId 표기(지원 문의용) |
- 사용자 메시지는 `error.message`를 그대로 노출 가능(서버 ko 문구, API_CONVENTIONS §8).

### 2.4 알림 (Toast)

- **성공 Toast**: 저장/삭제/업로드 완료/확정 완료(자동 3s 소멸).
- **경고 Toast/배너**: **예산 초과·경고**(DOMAIN_MODEL §2.4, `over`/`alert`) — 상태 색+아이콘+라벨.
- **진행 알림**: 업로드 잡 상태(폴링/SSE, API_CONVENTIONS §4) → 진행 중 배너, `completed` 시 성공 Toast + 검토 이동 링크.
- **낙관적 업데이트 롤백**(FRONTEND §8): 실패 시 Toast로 복구 통지.
- 접근성: Toast는 `role="status"`(aria-live=polite), 경고/에러는 `assertive`.

## 3. 반응형 · 접근성 상세

### 3.1 브레이크포인트 (Tailwind 기준, 모바일 우선)

| 토큰 | 폭 | 레이아웃 |
|------|----|---------|
| base | <640 | 단일 컬럼, **하단 탭 내비**(홈·거래·+·통계·설정), 차트 세로 스택 |
| `sm` 640 / `md` 768 | 태블릿 | 2컬럼 카드, 사이드바 접힘(아이콘) |
| `lg` 1024↑ | 데스크톱 | **좌측 사이드바 고정** + 다컬럼 그리드, 차트 병치 |

- **내비 전환**: `lg` 미만 = 하단 탭(FRONTEND §5), 이상 = 사이드바.
- **차트 반응형**: Recharts `ResponsiveContainer`, 좁은 폭에서 도넛→막대로 폼 강등 또는 라벨 외부 이동. 표 뷰는 가로 스크롤 컨테이너(`overflow-x:auto`).
- **빠른 입력(+)**: 모바일=바텀시트, 데스크톱=다이얼로그.

### 3.2 접근성 체크리스트 (a11y)

- **키보드**: 모든 인터랙티브 요소 tab 도달·포커스 링 가시, 다이얼로그 포커스 트랩+Esc 닫기, 표 정렬 헤더 키보드 조작.
- **대비**: 텍스트/아이콘 WCAG AA(4.5:1, 큰 텍스트 3:1). 라이트·다크 각각 충족. 수입/지출 색도 텍스트 대비 확보.
- **색 비의존**: 상태·계열을 색 단독으로 구분 금지 — 아이콘/라벨/텍스처 병행(§1.5·§2.4).
- **스크린리더**: 차트=표 대체+`aria-label`(요약), 아이콘 버튼 `aria-label`, 폼 `label` 연결·에러 `aria-describedby`.
- **모션**: `prefers-reduced-motion` 존중(차트 애니메이션·shimmer 축소).
- **터치 타깃**: 최소 44×44px(하단 탭·행 액션).
- **폼**(FRONTEND §9): zodResolver 인라인 에러, 제출 중 버튼 비활성+로딩.

### 3.3 다크모드

- class 전략(FRONTEND §10). 차트 색은 §1.2대로 다크 표면 기준 별도 스텝(자동 반전 아님).
- 표면·텍스트·경계 토큰 라이트/다크 쌍 정의, 시스템 설정 감지 + 수동 토글.

## 5. 확정 필요 사항

1. **카테고리 팔레트 소스**: 디자인 시스템 기본 팔레트 채택 vs 브랜드 색 정의(검증기로 스냅).
2. **상위 N 접기 기준**: 도넛/추세에서 N=7 고정 vs 비중 임계(예: <3% 기타)로 동적.
3. **차트 라이브러리 확정**: Recharts 유지(FRONTEND §2) vs 접근성 강한 대안 필요 시 재검토.
4. **표 뷰 범위**: 전 차트 필수 vs 복잡 차트만.
